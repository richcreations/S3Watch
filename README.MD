# S3Watch вҖ” Smartwatch Firmware for Waveshare ESP32-S3-Touch-AMOLED-2.06

Custom smartwatch firmware built on ESP-IDF 5.x and LVGL 9, targeting the [Waveshare ESP32-S3-Touch-AMOLED-2.06](https://www.waveshare.com/wiki/ESP32-S3-Touch-AMOLED-2.06) development board.

## Features

- **Two watchfaces** вҖ” selectable via settings; Face 1: large stacked HH/MM with coloured digits; Face 2: horizontal HH:MM with subscript AM/PM and SS, full date (mm/dd/yyyy), and full weekday name
- **Background selector** вҖ” three swappable background images, independently configurable per session
- **12/24-hour mode** вҖ” AM/PM label shown/hidden automatically; control screen clock respects the setting
- **Wi-Fi + NTP time sync** вҖ” connects on demand, syncs via NTP, then immediately releases the radio to save power; re-syncs daily at 3 AM if Wi-Fi permission is enabled
- **Wi-Fi permission toggle** вҖ” master switch in the control screen; when off, the radio never activates (NTP sync suppressed)
- **Wi-Fi settings** вҖ” scan for networks, connect with on-screen keyboard, configure NTP server
- **Notifications** вҖ” notification list and pop-up UI; push source requires BLE (currently disabled вҖ” see Known Limitations)
- **Step counter** вҖ” daily step count screen with configurable goal and progress ring; sensor task currently disabled (see Known Limitations)
- **Settings menu** вҖ” brightness, display timeout, sound on/off, notification volume, step goal, set time (roller pickers), Wi-Fi, NTP server, watchface style, background, storage browser, flashlight, restore defaults
- **Control screen** вҖ” quick-access panel with 8-button grid: Brightness, Silence, Flashlight, Battery, Wi-Fi toggle, Settings
- **Flashlight** вҖ” full-screen white display at 100% brightness; swipe right to exit and restore previous brightness
- **Audio alerts** вҖ” WAV/MP3 notification tones played via ES8311 codec; synthesized fallback tone if file not found
- **Display power management** вҖ” configurable auto-off timeout, wake on PMU power key, light-sleep compatible PM locks
- **Persistent settings** вҖ” stored as JSON on a dedicated SPIFFS partition; survives reboot
- **RTC persistence** вҖ” PCF85063A hardware RTC with NVS-backed time persistence in case of power loss

## Power Management

The firmware is designed to maximise time spent in light sleep when the display is off, and to let dynamic frequency scaling (DFS) drop the CPU to 40 MHz between events when the display is on.

| Source | Display ON | Display OFF |
|---|---|---|
| LVGL task (`taskLVGL`) | 33 ms refresh (30 fps) | **suspended** via `vTaskSuspend()` |
| Display manager task | 50 ms poll | 1 s poll (PMU wake-button check) |
| Back-button task | 20 ms poll | **blocked indefinitely** вҖ” unblocked by display-on callback |
| AXP2101 PMU monitor | 2000 ms | 2000 ms |
| Audio codec / I2S | open only during playback | closed |
| Wi-Fi radio | off unless NTP sync | off |

**Measured behaviour** (via `CONFIG_PM_PROFILING`):

| State | Time in light sleep | Time at 40 MHz | Time at 240 MHz |
|---|---|---|---|
| Display OFF | **~99%** | вҖ” | <1% |
| Display ON, idle | 0% (blocked by lock) | **~71%** | ~26% |

CPU configured for DFS between 40 MHz and 240 MHz with light sleep enabled (`esp_pm_configure`). CPU is powered down in light sleep (`CONFIG_PM_POWER_DOWN_CPU_IN_LIGHT_SLEEP`).

A single `ESP_PM_NO_LIGHT_SLEEP` lock вҖ” held only while the display is active вҖ” gates sleep. The audio codec is opened on each playback and closed immediately after, so its I2S `APB_FREQ_MAX` lock isn't held continuously. The BSP's I2S RX (microphone) channel is left initialised but not enabled, since the firmware never records вҖ” an enabled I2S channel holds an APB lock even when idle.

> **Note:** The AXP2101 INT pin (GPIO 35) is held LOW by the PMIC when USB power is connected. Level-triggered GPIO wakeup on this pin causes a sleep/wake spin loop and WDT reset. GPIO wakeup from the PMU is therefore not used; the display manager polls the PMU at 1 s intervals instead.

## Hardware

| Component | Part | Interface |
|---|---|---|
| MCU | ESP32-S3R8 (dual-core LX7 @ 240 MHz, 8 MB PSRAM) | вҖ” |
| Display | 2.06" AMOLED 410Г—502, SH8601 driver | QSPI |
| Touch | FT3168 capacitive controller (FT5x06-compatible) | I2C |
| IMU | QMI8658 6-axis (accel + gyro) | I2C |
| RTC | PCF85063A | I2C |
| PMU | AXP2101 (ALDO rails, charging, power key) | I2C |
| Audio | ES8311 codec + speaker | I2S |
| Flash | 32 MB external QSPI | вҖ” |
| Battery | 3.7 V Li-ion via MX1.25 header | вҖ” |

> **Note on ALDO rails:** The AXP2101 ALDO1вҖ“4 rails are disabled by default on cold boot. The firmware explicitly enables them before `bsp_display_start()` to ensure the FT3168 touch controller has power. The BSP cuts the rails when the display sleeps and re-enables them on wake.

## Building and Flashing

### Prerequisites

- [Visual Studio Code](https://code.visualstudio.com/) with the [ESP-IDF Extension](https://marketplace.visualstudio.com/items?itemName=espressif.esp-idf-extension)
- ESP-IDF **v5.5.4** (install via the extension wizard or manually)
- USB-C cable connected to the board's USB port

### Steps

1. **Install the ESP-IDF extension** вҖ” open VS Code, go to Extensions, search for `ESP-IDF`, and install it. Run `ESP-IDF: Configure ESP-IDF extension` from the command palette and let it download ESP-IDF 5.5.4 and the toolchain.

2. **Open the project** вҖ” `File вҶ’ Open Folder...` вҶ’ select the `S3Watch` directory. The ESP-IDF status bar appears at the bottom of the window.

3. **Set the target** вҖ” click `Set Target` in the status bar (or `ESP-IDF: Set Target`) and choose `esp32s3`.

4. **Build** вҖ” click `Build Project` in the status bar or run `ESP-IDF: Build Project`. First build takes several minutes as managed components are fetched and compiled.

5. **Select the port** вҖ” connect the board via USB-C, then run `ESP-IDF: Select Serial Port` and pick the device (e.g., `/dev/cu.usbmodem*` on macOS, `COMx` on Windows).

6. **Flash** вҖ” run `ESP-IDF: Flash (UART)` to program the board, or `ESP-IDF: Flash and Monitor` to flash and immediately open the serial monitor.

## Configuration

Key options in [`sdkconfig.defaults`](sdkconfig.defaults):

| Option | Value | Purpose |
|---|---|---|
| `CONFIG_ESP_DEFAULT_CPU_FREQ_MHZ_240` | `y` | Run CPU at 240 MHz |
| `CONFIG_SPIRAM_MODE_OCT` | `y` | Octal PSRAM mode (8 MB) |
| `CONFIG_FREERTOS_HZ` | `200` | Lower tick rate to reduce idle overhead |
| `CONFIG_FREERTOS_TIMER_TASK_STACK_DEPTH` | `6144` | Extra stack for SPIFFS + cJSON in timer callbacks |
| `CONFIG_PM_ENABLE` | `y` | Dynamic frequency scaling + light sleep |
| `CONFIG_PM_POWER_DOWN_CPU_IN_LIGHT_SLEEP` | `y` | Power down CPU core during light sleep |
| `CONFIG_LV_DEF_REFR_PERIOD` | `33` | ~30 fps LVGL refresh |
| `CONFIG_ESP_WIFI_ENABLED` | `y` | Wi-Fi stack for NTP sync and settings |
| `CONFIG_BT_ENABLED` | `n` | BLE stack disabled |

## Known Limitations

- **BLE disabled** вҖ” `CONFIG_BT_ENABLED=n`; the `ble_sync` and `nimble-nordic-uart` components exist as stubs for future ANCS/NUS support. The notifications UI is present but receives no data without a BLE source.
- **Step counter disabled** вҖ” the QMI8658 sensor task is commented out in `main.cpp`; the steps screen shows but does not update.
- **Wi-Fi radio released after NTP sync** вҖ” the radio is stopped once time is synced to save power. Re-opening Wi-Fi settings wakes the radio on demand.
- **No VBAT backup for RTC** вҖ” the PCF85063A loses power when the main battery is fully drained; time is recovered from NVS on next boot but may be slightly stale.
- **PMU GPIO wakeup not used** вҖ” the AXP2101 INT pin (GPIO 35) is held LOW when USB power is connected, making level-triggered wakeup unsafe (causes WDT reset). The display manager polls instead.
- **Watchface width variability** вҖ” `font_numbers_120` has non-monospaced digits ('4' is 85 px wide vs '1' at 63 px). For digit combinations heavy in '4's, the time row may slightly overflow the 410 px screen width.
